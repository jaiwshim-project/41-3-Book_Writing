Claude Code로 책 집필하기

비개발자를 위한 실전 가이드 — 음성으로 말하고, AI가 만든다

이 가이드는 누구를 위한 것인가

코딩을 전혀 모르는 비개발자가 Claude Code를 이용하여 HTML 기반의 책을 집필하는 전 과정을 정리한 실전 가이드다. 실제로 바이브 코딩 책을 집필하면서 사용한 방법과 명령어, 시행착오를 그대로 담았다. 이 가이드를 따라하면, 당신도 자신만의 책을 직접 만들 수 있다.

전체 프로세스 한눈에 보기

단계 작업 설명
1환경 준비Claude Code 설치, 작업 폴더 생성, 음성 입력 설정
2챕터 파일 생성HTML 기본 구조와 레이아웃 설계
3콘텐츠 작성음성/텍스트로 내용 지시, 표·이미지·인용문 삽입
4레이아웃 최적화CSS 클래스화, 제목 계층 통일, 스타일 정리
5콘텐츠 품질 개선중복 제거, 분량 확장, 팩트 체크, 특별 디자인
6가독성 향상글씨 진하게, 줄 간격 조정, 렌더링 최적화
7다운로드 기능DOCX 변환, Word 호환 처리

Step 1. 환경 준비

1-1. Claude Code 설치

터미널(명령 프롬프트)을 열고 다음 명령어를 입력한다.

npm install -g @anthropic-ai/claude-code

설치가 완료되면, 작업할 폴더로 이동한 뒤 Claude Code를 실행한다.

cd C:\내책\집필
claude

1-2. 작업 폴더 구성

집필에 사용할 폴더를 미리 만들어둔다. 챕터 파일, 이미지, 스크린샷을 한 곳에 모아두면 관리가 편하다.

내책/
├── 집필/
│   ├── chapter1.html
│   ├── chapter2.html
│   ├── guide.html
│   ├── 마이크.jpg
│   ├── 타로.jpg
│   └── 운세.jpg

1-3. 음성 입력 활용 (Win+H)

Claude Code에게 명령을 내릴 때 타이핑 대신 음성 입력을 사용하면 훨씬 빠르고 자연스럽다.

  1. Windows 키 + H를 동시에 누르면 음성 입력 팝업이 뜬다
  2. 마이크 아이콘이 활성화되면 말하기 시작한다
  3. 말을 멈추면 자동으로 텍스트로 변환된다
  4. 변환된 텍스트를 확인하고 Enter를 누른다
음성 입력 팁

완벽한 문장이 아니어도 된다. "챕터2에서 소확행 아이템 설명을 표로 만들어" 같은 자연스러운 말투가 가장 효과적이다. Claude Code는 문맥을 이해하기 때문에, 키워드 중심으로 말해도 충분하다.

Step 2. 챕터 HTML 파일 생성

2-1. 첫 챕터 파일 만들기

Claude Code에게 첫 챕터 파일을 만들어달라고 요청한다.

"chapter1.html 파일을 만들어. A4 사이즈에 맑은 고딕 폰트로, 책 형식으로 만들어줘"
Claude Code가 HTML 기본 구조, CSS 스타일, 본문 영역을 포함한 파일을 생성한다

2-2. 기본 레이아웃 요소

책 형식의 HTML에 필요한 기본 레이아웃 요소들이다. 처음 파일을 만들 때 이 요소들을 포함시킨다.

순서 요소 설명
1A4 페이지 설정@page { size: A4; margin: 20mm; } — 인쇄 시 A4 규격 적용
2폰트맑은 고딕 기본, 본문 14px, 줄 간격 1.8
3본문 너비max-width: 800px — 읽기 편한 너비로 제한
4여백margin: 40px auto; padding: 40px; — 중앙 정렬 + 내부 여백
5DOCX 다운로드 버튼우측 상단 고정 버튼 — 클릭 시 Word 파일로 변환

2-3. 제목 계층 설계

책의 가독성을 높이는 핵심은 제목 계층의 시각적 구분이다. 색상과 스타일로 h1, h2, h3를 명확하게 분리한다.

순서 제목 스타일 용도
1h1 (대제목)다크 네이비 #0d0d1a, 24px, 중앙 정렬챕터 제목
2h2 (중제목)인디고 블루 #1e2d5e, 18px, 하단 선주요 섹션 제목
3h3 (소제목)다크 그레이 #1a1a1a, 15px, 좌측 컬러 바세부 항목 제목
"제목 계층을 만들어. h1은 다크 네이비에 중앙 정렬, h2는 인디고 블루에 하단 선, h3는 좌측에 파란 바를 넣어줘"

Step 3. 콘텐츠 작성

이 단계가 집필의 핵심이다. 음성이나 텍스트로 Claude Code에게 내용을 지시하면, AI가 HTML로 작성해준다.

3-1. 텍스트 추가하기

새로운 내용을 추가할 때는 어디에, 무엇을 넣을지 명확하게 말한다.

"챕터2에서 클라라 라디오카페 섹션 아래에 소확행 아이템 설명을 추가해"
"Step 1과 Step 2 사이에 음성 입력 사용법 설명을 넣어줘"
핵심 원칙: 위치 + 내용

"어디에" + "무엇을"만 말하면 된다. Claude Code가 적절한 HTML 태그와 서식을 알아서 적용한다. "p 태그로 감싸고 margin-top 10px 넣어줘" 같은 코딩 용어를 쓸 필요가 없다.

3-2. 텍스트 수정하기

이미 작성된 내용을 수정할 때는 무엇을 어떻게 바꿀지 말한다.

"2월 15일 기준 총 45개를 2월 10일 기준 총 40개로 수정해"
"41일 동안 40개를 40일 동안 40개로 바꿔"
"실시간 라디오 스트리밍 설명을 '국내 8개 방송을 웹에서 바로 재생'으로 변경해"

3-3. 콘텐츠 삭제하기

"프리셋 저장 항목은 삭제해"
"AI 음악 생성 기능 삭제하고, 비주얼라이저는 이퀄라이저로 대체해"

3-4. 표(Table) 만들기

정보를 정리할 때 표는 가장 효과적인 도구다.

"소확행 아이템 9개를 표로 만들어. 순서, 아이템 이름, 기능 설명 3개 컬럼으로"
"8가지 플랫폼을 순서, 플랫폼명, 분야, 핵심 기능 수, 핵심 교훈으로 정리해"
Claude Code가 thead/tbody 구조의 테이블을 자동 생성하고, 짝수 행 배경색, 호버 효과까지 CSS로 적용한다

3-5. 블럭쿼트(인용/팁/주의) 활용

강조하고 싶은 내용은 블럭쿼트로 시각적으로 분리한다. 4가지 유형을 상황에 맞게 사용한다.

순서 클래스 색상 용도
1.bq-info파란색 좌측 바 + 연파랑 배경정보, 안내, 배경 설명
2.bq-tip노란색 좌측 바 + 연노랑 배경팁, 기획의 핵심, 조언
3.bq-warn빨간색 좌측 바 + 연빨강 배경주의사항, 경고
4.bq-success초록색 좌측 바 + 연초록 배경성공 사례, 결론, 핵심 메시지
"기획의 핵심을 노란색 팁 박스로 만들어"
"주의할 점을 빨간색 경고 박스에 넣어줘"

3-6. 이미지 삽입

스크린샷이나 사진을 추가할 때는 파일 경로를 알려주면 된다.

"C:\Users\USER\Downloads\타로.jpg 이미지를 디지털 화면 섹션에 추가해"
"마이크.jpg와 운세.jpg 이미지를 나란히 넣어줘"
Claude Code가 이미지를 프로젝트 폴더로 복사하고, 적절한 크기와 캡션과 함께 HTML에 삽입한다

3-7. 외부 사이트 참고하여 작성

실제 사이트의 내용을 참고하여 정확한 정보를 작성할 수 있다.

"https://jaiwshim-project.github.io/37-RadioCafe/manual.html 이 자료 활용하여 업데이트해"
Claude Code가 해당 URL의 내용을 읽고, 매뉴얼에 있는 실제 기능 디테일(카테고리 5종, 엽서 디자인 4종, 타임캡슐 최대 1년 등)을 반영하여 설명을 업데이트한다

Step 4. 레이아웃 최적화

콘텐츠 작성이 어느 정도 진행되면, 레이아웃을 정리할 때가 온다. 이 단계에서는 코드를 깔끔하게 만들고, 일관된 디자인을 적용한다.

4-1. 인라인 스타일 → CSS 클래스 전환

콘텐츠를 추가하다 보면 인라인 스타일(style="...")이 곳곳에 쌓인다. 이것을 CSS 클래스로 전환하면 유지보수가 훨씬 쉬워진다.

"챕터2에 적용한 것처럼 챕터1도 최적화시켜. 인라인 스타일을 CSS 클래스로 전환해"

변환 전 (인라인 스타일):

<blockquote style="border-left:4px solid #2563eb; padding:8px 16px; background:#eff6ff;">

변환 후 (CSS 클래스):

<blockquote class="bq-info">

4-2. 제목 색상 계층 통일

"모든 h2의 인라인 스타일을 제거하고 CSS로 통일해. h2는 인디고 블루, h3는 다크 그레이로"

4-3. 재사용 가능한 CSS 클래스 목록

집필 과정에서 반복적으로 사용하게 되는 클래스들이다.

순서 클래스 용도
1.bq-info / .bq-tip / .bq-warn / .bq-success블럭쿼트 4가지 유형
2.section-divider섹션 간 굵은 구분선
3.sub-divider섹션 내 가는 구분선
4.content-wrap본문 감싸기 (line-height, font-size)
5.cmd-dark / .cmd-light코드 블록 (다크/라이트)
6code.inline본문 내 인라인 코드
7.badge-edu / .badge-biz / .badge-creative / .badge-health분야별 배지 (파랑/빨강/보라/초록)
8.screenshot-note이미지 아래 캡션 텍스트

Step 5. 콘텐츠 품질 개선

5-1. 중복 내용 제거 및 병합

집필 중 같은 내용이 다른 위치에 반복되는 경우가 생긴다. Claude Code에게 중복을 찾아 병합하도록 요청한다.

"챕터1에서 Step 1 개발 환경 준비하기 내용이 맨 앞에도 있어서 중복이야. 앞쪽 내용과 합쳐서 하나로 만들어"
"기획의 핵심 내용이 앞에서 나온 것이 뒤에서 반복되는 경우가 있어. 새롭고 도움 되는 내용으로 변경해"

5-2. 콘텐츠 확장 (분량 늘리기)

특정 섹션의 분량을 늘리고 싶을 때는 배수를 지정하면 된다.

"8가지 플랫폼에서 발견한 공통점 내용을 세 배 확장해"
기존 5개 항목이 15개 항목으로 확장되며, 각 항목에 실제 플랫폼 사례가 구체적으로 연결된다

5-3. 구조 변경

섹션을 분리하거나 통합할 때도 자연스럽게 요청한다.

"지금 추가한 10번~12번은 별도로 분리해. 디지털 화면이라는 제목 아래에 넣고, 위에 1번~9번은 아날로그 화면으로 수정해"

5-4. 특별 디자인 박스 만들기

중요한 내용을 시각적으로 돋보이게 만들고 싶을 때는 디자인 요소를 요청한다.

"지금 추가한 것을 박스 안에 넣고, 연한 분홍색 배경색을 넣어서 눈에 띄게 만들어"
분홍색 그라데이션 배경, 둥근 모서리, 그림자 효과, 내부 흰색 카드 레이아웃이 적용된 특별 박스가 생성된다

5-5. 법적 보호 표기 등 특수 문구

"클라라 라디오카페와 파이어내비는 특허와 저작권 등록으로 법적 보호를 받고 있다는 표기를 해. 빨간색 글씨로 눈에 띄게"

Step 6. 가독성 향상

6-1. 글씨 진하게 만들기

화면에서 글씨가 연하게 보이면 색상 투명도를 조정한다.

"챕터1과 챕터2에 있는 모든 글씨의 투명도를 높여서 진하게 보이게 만들어"

Claude Code가 처리하는 것:

순서 요소 변경 전 변경 후
1본문(body)#222 (연한 검정)#000 (완전한 검정)
2소제목(h3)#555#1a1a1a
3블럭쿼트#444#111
4부제/설명#666#333
5캡션/푸터#888#444

6-2. 폰트 렌더링 최적화

다음 CSS 속성을 body에 추가하면 글자가 더 선명하게 렌더링된다.

-webkit-font-smoothing: antialiased;
-moz-osx-font-smoothing: grayscale;
text-rendering: optimizeLegibility;

6-3. 줄 간격(line-height) 조정

한글 텍스트는 line-height: 1.8 이상이 읽기 편하다. 기본값(1.6)보다 넉넉하게 설정한다.

Step 7. DOCX 다운로드 기능

7-1. 왜 DOCX 다운로드가 필요한가

HTML로 집필하면 브라우저에서는 예쁘게 보이지만, 출판사에 전달하거나 오프라인에서 편집하려면 Word 파일이 필요하다. html-docx-js 라이브러리를 사용하면 HTML을 그대로 DOCX로 변환할 수 있다.

7-2. 기본 구현

"DOCX 다운로드 버튼을 우측 상단에 고정으로 만들어줘"

7-3. WYSIWYG 유지 — 핵심 문제와 해결

DOCX로 변환할 때 가장 큰 문제는 CSS 클래스 스타일이 Word에서 사라지는 것이다.

문제: 왜 서식이 사라지는가

Word는 <style> 태그의 CSS 클래스를 인식하지 못한다. class="bq-info"로 적용한 스타일은 Word에서 완전히 무시된다. Word가 인식하는 것은 인라인 스타일(style="...")뿐이다.

해결: 다운로드 전 인라인 스타일로 변환

다운로드 버튼을 클릭하면, JavaScript가 모든 요소의 getComputedStyle()을 읽어서 인라인 스타일로 변환한 뒤 DOCX를 생성한다. 브라우저에서 보이는 그대로 Word에서도 보인다.

7-4. Word 미지원 CSS 호환 처리

인라인 스타일로 변환해도 Word가 지원하지 않는 CSS 속성이 있다. 이것을 자동으로 호환 형식으로 바꿔주는 wordifyClone() 함수가 필요하다.

순서 Word 미지원 CSS 호환 변환
1display: flex (카드 가로 배치)<table> 가로 셀로 변환
2linear-gradient (그라데이션)단색 background-color로 대체
3border-radius (둥근 모서리)제거
4box-shadow (그림자)제거
5max-width (이미지)고정 width로 변환
"DOCX 다운로드할 때 위지윅이 사라져. 해결해줘"
Claude Code가 inlineAllStyles()와 wordifyClone() 함수를 추가하여 Word 호환 DOCX를 생성한다

Step 8. 실전에서 자주 쓰는 명령 패턴

30"DOCX 다운로드 버튼을 클릭해서 파일을 오픈하니까 위지윅이 사라져버리네. 이거 해결해"DOCX 서식 유지 문제 해결 — HTML→DOCX 변환 시 CSS 클래스 스타일이 사라지는 문제를 보고한다. Claude Code가 inlineAllStyles() 함수를 추가하여 모든 스타일을 인라인으로 변환한 뒤 DOCX를 생성하도록 수정한다.
31"다운로드 결과물.jpg와 HTML 화면.jpg — 두 가지가 서로 달라. 확인해"스크린샷 비교 디버깅 — HTML 화면과 다운로드된 DOCX의 스크린샷을 비교하여 차이점을 확인한다. Claude Code가 이미지를 분석하여 flex→세로 풀림, 그라데이션 사라짐, 왼쪽 바 사라짐 등 구체적인 문제를 파악하고 wordifyClone() 함수로 Word 호환 변환을 추가한다.

가이드 제작

순서 명령어 목적 및 용도
32"이 과정을 처음부터 지금까지 한 과정을 정리하고 싶어. 다른 사람들이 나처럼 Claude Code를 이용해서 집필할 때 가이드북으로 만들어줘. guide.html로 만들어"가이드북 제작 — 전체 집필 과정을 체계적으로 정리한 가이드 HTML을 생성한다. 동일한 CSS 스타일, DOCX 다운로드 기능을 포함하여 독립적인 문서로 완성한다.
33"내가 집필하면서 사용했던 명령어들을 순서대로 쭉 모아줘. 각 명령마다 목적과 용도 설명을 추가해"명령어 로그 정리 — 실제 사용한 모든 명령어를 시간순으로 정리하고, 각 명령의 의도와 효과를 문서화한다. 다른 사용자가 참고할 수 있는 실전 레퍼런스가 된다.
명령어 로그에서 발견할 수 있는 패턴
  1. 자연어 중심: 33개 명령어 중 코딩 용어를 사용한 것은 거의 없다. "박스 넣어줘", "빨간색으로", "세 배 확장해" 같은 일상적인 표현이 대부분이다.
  2. 점진적 완성: 한 번에 완성하지 않는다. 기본 콘텐츠 → 구조 정리 → 디자인 보강 → 가독성 조정 → 문제 해결 순서로 단계적으로 발전시킨다.
  3. 즉각적 피드백: 결과를 확인하고 바로 수정을 요청한다. "투명도 높여" → 확인 → "더 높여" 같은 반복 조정이 자연스럽다.
  4. 이미지로 소통: 문제를 설명하기 어려울 때 스크린샷 2장을 보여주며 "이 두 가지가 달라. 확인해"라고 말하는 것이 가장 효과적이다.
  5. URL로 팩트 체크: 실제 사이트 URL을 주고 "이 자료 활용하여 업데이트해"라고 말하면, AI가 사이트를 읽고 정확한 정보로 재작성한다.

집필을 마치며 — 핵심 요약

Claude Code 집필의 5가지 원칙
  1. 코딩 용어를 쓰지 마라. "h2 태그에 color 속성을 변경해줘" 대신 "중간 제목을 파란색으로 바꿔줘"라고 말하면 된다. 자연스러운 말이 가장 정확한 명령이다.
  2. 위치를 명확히 말하라. "그 내용 수정해"보다 "챕터2 클라라 라디오카페 섹션에서 소확행 아이템 표를 수정해"가 훨씬 정확한 결과를 낸다.
  3. 한 번에 하나씩 요청하라. 여러 가지를 한꺼번에 말하면 누락이 생길 수 있다. "표 만들어줘" → 확인 → "이미지 추가해" → 확인 → "박스로 감싸줘" 순서가 안전하다.
  4. 결과를 즉시 확인하라. HTML 파일을 브라우저에서 열어두고, 수정이 반영될 때마다 새로고침(F5)해서 확인한다. 문제가 있으면 바로 수정을 요청한다.
  5. 음성 입력을 적극 활용하라. 타이핑보다 말하는 것이 3배 빠르다. Win+H로 음성 입력을 켜고, 자연스럽게 말하면 된다. 완벽한 문장이 아니어도 Claude Code는 의도를 파악한다.

당신의 지식과 경험이 곧 책의 원재료다.
Claude Code는 그 원재료를 아름다운 책으로 빚어주는 도구다.
