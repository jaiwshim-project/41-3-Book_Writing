17장. 바이브 코딩 — 코딩이 아니라, 대화다

바이브 코딩이란 무엇인가

"바이브 코딩(Vibe Coding)"이라는 말은 2025년 2월, OpenAI의 공동 창업자 안드레이 카르파시(Andrej Karpathy)가 처음 사용했다. 그는 자신의 소셜 미디어에 이렇게 썼다.

안드레이 카르파시의 원문 (의역)

"나는 요즘 완전히 새로운 종류의 코딩을 하고 있다. 나는 그것을 '바이브 코딩'이라고 부른다. 코드를 보지 않고, 분위기(vibe)에 따라 지시하고, AI가 만든 결과를 실행해보고, 되면 넘어간다. 코드가 맞는지 틀린지도 잘 모르겠고, 그냥 느낌(vibe)으로 한다."

이 말이 등장하자마자 전 세계 개발자 커뮤니티가 들끓었다. 찬사와 비판이 동시에 쏟아졌다. "코드를 이해하지 않고 프로그래밍을 한다고?" "그건 개발이 아니다!" 하지만 동시에, 비개발자들 사이에서는 새로운 가능성에 대한 흥분이 번졌다. "나도 프로그램을 만들 수 있다고?"

바이브 코딩의 핵심은 간단하다.

전통적 코딩: 프로그래밍 언어를 배운다 → 문법에 맞게 코드를 작성한다 → 오류를 수정한다 → 실행한다

바이브 코딩: AI에게 자연어로 말한다 → AI가 코드를 만든다 → 결과를 확인한다 → 수정을 요청한다

이것은 단순한 도구의 변화가 아니다. 코딩이라는 행위의 본질이 바뀐 것이다. 이전에는 "어떻게 만드는가(How)"를 알아야 했다면, 이제는 "무엇을 만들 것인가(What)"만 알면 된다.

바이브 코딩은 왜 가능해졌는가

바이브 코딩이 가능해진 데에는 세 가지 기술적 기반이 있다.

순서 기술 기반 의미
1대규모 언어 모델 (LLM)자연어를 이해하고 코드를 생성할 수 있는 AI가 등장했다. Claude, GPT 등이 수십억 줄의 코드를 학습하여, 사람의 말을 코드로 번역할 수 있게 되었다.
2컨텍스트 윈도우 확대AI가 한 번에 처리할 수 있는 텍스트 양이 폭발적으로 늘어났다. 전체 프로젝트의 맥락을 이해한 상태에서 코드를 만들 수 있게 되었다.
3실행 환경 통합AI가 코드를 만드는 것에 그치지 않고, 직접 파일을 생성하고, 서버에 배포하고, 오류를 수정하는 것까지 하나의 흐름으로 가능해졌다.

이 세 가지가 결합되면서, "말로 프로그래밍한다"는 것이 꿈이 아닌 현실이 되었다.

바이브 코딩과 전통적 코딩의 결정적 차이

구분 전통적 코딩 바이브 코딩
진입 장벽프로그래밍 언어 학습 필수 (수개월~수년)자연어(한국어, 영어)만 알면 시작 가능
작업 방식코드를 직접 타이핑AI에게 말로 지시
오류 수정디버깅 기술 필요"이 부분이 안 돼. 고쳐줘"라고 말하면 됨
개발 속도기능 하나에 수시간~수일기능 하나에 수분~수십분
결과물 수준개발자 역량에 의존AI 모델 성능 + 지시의 명확성에 의존
적합한 사람개발자, 컴퓨터공학 전공자아이디어가 있는 모든 사람

바이브 코딩의 한계도 알아야 한다

바이브 코딩이 마법은 아니다. 솔직하게 한계를 알아야, 적절하게 활용할 수 있다.

  1. 복잡한 비즈니스 로직에는 한계가 있다. 단순한 웹사이트, 정보 서비스, CRUD 애플리케이션에는 탁월하지만, 금융 시스템이나 대규모 트래픽 처리 같은 복잡한 시스템은 전문 개발자가 필요하다.
  2. AI가 완벽하지 않다. 가끔 틀린 코드를 만들고, 이전에 고친 것을 다시 되돌리기도 한다. 하지만 다시 지시하면 된다. 그 과정 자체가 바이브 코딩이다.
  3. '무엇을 만들지'를 아는 것이 핵심이다. AI는 "만들어줘"라는 지시에 반응한다. 무엇을 만들어야 하는지를 모르면, AI도 만들 수 없다. 도메인 지식과 사업 경험이 있는 사람이 바이브 코딩에서 압도적으로 유리한 이유가 여기에 있다.
시니어에게 바이브 코딩이 유리한 이유

바이브 코딩에서 가장 중요한 것은 코딩 기술이 아니라, "무엇을 만들어야 하는지"를 아는 것이다. 30년간 현장에서 문제를 발견하고, 고객을 만나고, 시장을 이해해온 시니어는 "무엇을"에 대한 답을 이미 가지고 있다. 젊은 개발자에게 없는 것은 경험이다. 시니어에게 없었던 것은 도구였다. 이제 도구가 나왔다.

18장. 클로드 코드 — 당신의 말을 코드로 바꾸는 AI

클로드 코드란 무엇인가

클로드 코드(Claude Code)는 미국 AI 기업 앤트로픽(Anthropic)이 만든 AI 코딩 도구다. 터미널(명령줄)에서 작동하며, 자연어로 지시하면 코드를 작성하고, 파일을 만들고, 오류를 수정하고, 서버에 배포하는 것까지 해준다.

비유하자면, 클로드 코드는 "24시간 대기하는 개인 개발자"다. 밤 11시에 "이 버튼 색깔 바꿔줘"라고 말해도, 3초 안에 바꿔준다. 인건비도, 대기 시간도, 커뮤니케이션 비용도 없다.

클로드 코드는 어떻게 작동하는가

클로드 코드의 작동 방식은 놀랍도록 단순하다.

당신이 말한다 → 클로드가 이해한다 → 코드를 만든다 → 파일을 저장한다 → 실행한다

예를 들어, 이렇게 말하면:

"카페 메뉴판 홈페이지를 만들어줘. 아메리카노 4,500원, 카페라떼 5,000원, 녹차라떼 5,500원을 넣어줘. 예쁜 디자인으로."

클로드 코드는 이 한 문장으로 HTML 파일을 만들고, CSS로 디자인하고, 이미지 배치까지 해서 완성된 홈페이지를 내놓는다. 코드를 한 줄도 쓰지 않았다. 말만 했다.

클로드 코드가 할 수 있는 것들

순서 기능 설명
1코드 작성HTML, CSS, JavaScript, Python, React 등 거의 모든 프로그래밍 언어로 코드를 작성한다
2파일 관리파일을 생성, 수정, 삭제한다. 폴더 구조도 만들어준다
3오류 수정에러가 발생하면 원인을 분석하고 자동으로 수정한다
4검색 및 참조프로젝트 내 모든 파일을 읽고 이해한 뒤, 맥락에 맞는 코드를 만든다
5Git 관리코드 변경 이력을 관리하고, GitHub에 업로드한다
6배포만든 서비스를 인터넷에 공개(배포)한다
7리팩토링기존 코드를 더 깔끔하고 효율적으로 개선한다

클로드 코드 사용료

클로드 코드를 사용하려면 앤트로픽의 유료 구독이 필요하다.

구분 요금제 월 비용 특징
1Pro$20 (약 2.8만원)일반 사용에 충분. 하루 사용량 제한 있음
2Max 5x$100 (약 14만원)Pro 대비 5배 사용량. 집중 개발 시 권장
3Max 20x$200 (약 28만원)Pro 대비 20배 사용량. 전일 개발자용
비용 관점에서 생각해보자

개발자를 고용하면 월 최소 400만~700만 원이다. 프리랜서에게 맡겨도 프로젝트당 1,000만~5,000만 원이 든다. 클로드 코드 Max 요금제(월 14만 원)로 혼자서 하루에 하나씩 서비스를 만들 수 있다면, 이것은 비용이 아니라 투자다.

클로드 코드 설치 과정

클로드 코드를 설치하려면 몇 가지 사전 프로그램이 필요하다. 복잡해 보이지만, PART 1 실습에서 이미 경험한 과정이다.

1단계. Node.js 설치

클로드 코드가 작동하기 위해 필요한 기반 프로그램이다. nodejs.org에서 LTS 버전을 다운로드하여 설치한다. "다음 → 다음 → 설치 완료"로 끝난다.

2단계. 클로드 코드 설치

터미널(또는 명령 프롬프트)을 열고 다음 한 줄을 입력한다.

npm install -g @anthropic-ai/claude-code

설치가 완료되면, claude라고 입력하면 클로드 코드가 실행된다.

3단계. 로그인

처음 실행하면 앤트로픽 계정으로 로그인하라는 안내가 나온다. 웹 브라우저가 열리고, 로그인하면 터미널과 연결된다.

4단계. 작업 폴더에서 시작

작업할 폴더로 이동한 뒤, claude를 입력하면 바로 대화가 시작된다. 이때부터 "홈페이지 만들어줘"라고 말하면 된다.

클로드 코드를 잘 쓰는 비결

30년간 교육과 컨설팅을 하면서 배운 것이 하나 있다. 좋은 질문이 좋은 답을 만든다. 클로드 코드도 마찬가지다.

구분 이렇게 말하면 이렇게 말하면 더 좋다
모호함"홈페이지 만들어줘""카페 소개 홈페이지를 만들어줘. 메뉴판, 오시는 길, 영업시간이 있고, 따뜻한 느낌의 디자인으로"
수정"이거 좀 바꿔줘""제목 글씨를 24px로 키우고, 색상을 남색(#1e2d5e)으로 바꿔줘"
추가"기능 추가해줘""메뉴판 아래에 '주문하기' 버튼을 추가하고, 클릭하면 카카오톡 채널로 연결되게 해줘"
참고"이 사이트처럼 해줘""이 URL의 레이아웃을 참고해서, 우리 카페 정보로 바꿔서 만들어줘"
음성 입력의 힘

Windows에서는 Win + H 키를 누르면 음성 인식이 시작된다. 키보드를 치지 않고, 말로 지시할 수 있다. 이 책의 제목이 "말로 만드는 사업"인 이유가 여기에 있다. 실제로 필자는 이 책의 모든 챕터를 음성으로 지시하여 작성했다.

19장. GitHub — 당신이 만든 것을 보관하고 공개하는 곳

GitHub이란 무엇인가

GitHub(깃허브)는 전 세계 1억 명 이상의 개발자가 사용하는 코드 저장 및 공유 플랫폼이다.

비유하자면, GitHub는 "코드의 구글 드라이브"다. 구글 드라이브에 문서를 저장하듯, GitHub에 코드를 저장한다. 다만, GitHub에는 구글 드라이브에 없는 결정적인 기능이 하나 더 있다 — 버전 관리다. 파일을 수정할 때마다 "누가, 언제, 무엇을" 바꿨는지를 자동으로 기록한다.

비개발자에게 GitHub이 필요한 이유

"나는 개발자가 아닌데, GitHub이 왜 필요한가?" 바이브 코딩에서 GitHub은 세 가지 역할을 한다.

순서 역할 왜 필요한가
1저장소클로드 코드가 만든 코드를 안전하게 보관한다. 컴퓨터가 고장 나도 코드는 살아있다.
2배포 연결GitHub에 코드를 올리면, Vercel이 자동으로 인터넷에 공개해준다. GitHub이 없으면 배포가 불가능하다.
3이력 관리언제든 이전 버전으로 돌아갈 수 있다. "어제 수정하기 전이 더 나았는데..." 할 때 되돌릴 수 있다.

GitHub 핵심 용어 — 이것만 알면 된다

GitHub에는 수많은 용어가 있지만, 바이브 코딩에서 필요한 것은 딱 5개다.

순서 용어 비유 실제 의미
1Repository (저장소)프로젝트 폴더하나의 프로젝트에 대한 모든 파일을 담는 공간
2Commit (커밋)저장 버튼변경사항을 기록하는 행위. "로그인 기능 추가"처럼 메모를 남긴다
3Push (푸시)클라우드에 업로드내 컴퓨터의 변경사항을 GitHub 서버로 올리는 것
4Clone (클론)다운로드GitHub에 있는 프로젝트를 내 컴퓨터로 가져오는 것
5Branch (브랜치)작업 사본원본을 건드리지 않고 따로 작업하는 공간. 완성되면 합친다
실제로는 이렇게 쓴다

걱정하지 마라. 바이브 코딩에서는 이 용어들을 외울 필요가 없다. 클로드 코드에게 "깃허브에 올려줘"라고 말하면, 커밋과 푸시를 알아서 해준다. 용어는 "아, 이게 그거였구나" 정도로 알면 충분하다.

GitHub 요금제

순서 요금제 월 비용 특징
1Free무료무제한 공개/비공개 저장소. 개인 사용에 충분
2Pro$4 (약 5,600원)고급 인사이트, 위키 기능
3Team$4/인 (약 5,600원)팀 관리, 보호된 브랜치

무료

개인 바이브 코딩에는 Free 요금제로 충분하다. 비용 걱정 없이 시작할 수 있다.

GitHub 가입 및 시작

1단계. 가입

github.com에 접속하여 이메일로 회원가입한다. 무료 계정이면 충분하다.

2단계. Git 설치

GitHub를 사용하려면 컴퓨터에 Git이라는 프로그램이 설치되어 있어야 한다. Windows에서는 터미널에 winget install Git.Git을 입력하면 설치된다.

3단계. GitHub CLI 설치

GitHub CLI(gh)를 설치하면 터미널에서 바로 GitHub을 제어할 수 있다. winget install GitHub.cli로 설치하고, gh auth login으로 로그인한다.

4단계. 클로드 코드와 연결

이후 클로드 코드에게 "깃허브에 새 저장소 만들고 올려줘"라고 말하면 된다. 나머지는 클로드가 처리한다.

20장. Supabase — 데이터를 저장하고 관리하는 백엔드

Supabase란 무엇인가

수파베이스(Supabase)는 PostgreSQL 기반의 오픈소스 백엔드 플랫폼이다. 데이터베이스, 사용자 인증, 파일 저장소, 실시간 기능을 하나의 서비스에서 제공한다.

비유하자면, Supabase는 "보이지 않는 사무실"이다. 웹사이트가 앞에서 고객을 맞이하는 매장이라면, Supabase는 뒤에서 회원 정보를 관리하고, 주문을 기록하고, 파일을 보관하는 백오피스다. 고객 눈에는 보이지 않지만, 없으면 사업이 돌아가지 않는다.

언제 Supabase가 필요한가

모든 프로젝트에 Supabase가 필요한 것은 아니다. 구분하는 기준은 간단하다.

상황 Supabase 필요 예시
정보를 보여주기만 하면 될 때필요 없음회사 소개 페이지, 포트폴리오, 메뉴판
사용자가 데이터를 입력/저장해야 할 때필요함회원가입, 게시판, 예약 시스템
사용자별로 다른 화면을 보여줘야 할 때필요함마이페이지, 내 주문 내역, 대시보드
파일을 업로드/다운로드해야 할 때필요함프로필 사진, 문서 공유, 이미지 갤러리
PART 1 실습에서는 Supabase를 사용하지 않았다

2시간 실습에서 만든 홈페이지는 "정보를 보여주기만 하는" 정적 사이트였기 때문에 Supabase가 필요 없었다. 사용자 로그인이나 데이터 저장이 필요한 서비스를 만들 때부터 Supabase가 등장한다.

Supabase가 제공하는 기능

순서 기능 쉽게 말하면 활용 예시
1데이터베이스엑셀 같은 데이터 저장소 (하지만 훨씬 강력)회원 정보, 상품 목록, 주문 내역
2인증 (Auth)로그인/회원가입 시스템이메일, 구글, 카카오 소셜 로그인
3스토리지파일 보관함프로필 사진, 문서, 이미지 업로드
4실시간데이터 변경 시 즉시 화면 반영실시간 채팅, 알림, 대시보드
5Edge Functions서버 없는 백엔드 로직결제 처리, 이메일 발송, AI API 호출
6벡터 DBAI 검색을 위한 데이터 저장문서 Q&A, 유사 콘텐츠 추천

Supabase 요금제

순서 요금제 월 비용 특징
1Free무료DB 500MB, 스토리지 1GB, 2개 프로젝트. 개발/테스트에 충분
2Pro$25 (약 3.5만원)DB 8GB, 스토리지 100GB, 무제한 프로젝트. 실서비스용
3Team$599 (약 84만원)대규모 서비스, PITR 백업

무료

MVP(최소 기능 제품)를 만들고 시장을 테스트하는 단계에서는 Free 요금제로 충분하다.

Supabase 시작하기

1단계. 가입

supabase.com에 접속하여 GitHub 계정으로 회원가입한다.

2단계. 프로젝트 생성

"New Project"를 클릭하고, 프로젝트 이름을 입력한다. 리전(Region)은 Northeast Asia (Seoul)을 선택한다. 서울에 서버가 있으면 한국 사용자에게 빠르게 응답한다.

3단계. API 키 확인

프로젝트가 생성되면 Project Settings → API에서 URL과 API 키를 확인한다. 이 키를 프로젝트에 넣으면 Supabase와 연결된다.

4단계. 클로드 코드와 연결

클로드 코드에게 "이 Supabase 프로젝트와 연결해줘"라고 말하고 URL과 키를 알려주면, 나머지 설정을 알아서 해준다.

API 키 보안 주의

Supabase의 service_role 키는 절대 공개하면 안 된다. 이 키가 노출되면 누구나 당신의 데이터베이스에 접근할 수 있다. 클로드 코드에게 "이 키를 환경변수로 안전하게 관리해줘"라고 말하면 적절하게 처리해준다.

21장. Vercel — 만든 것을 세상에 공개하는 무대

Vercel이란 무엇인가

버셀(Vercel)은 웹 애플리케이션을 인터넷에 공개(배포)하는 클라우드 플랫폼이다. GitHub에 코드를 올리면, Vercel이 자동으로 인터넷에 공개해준다.

비유하자면, Vercel은 "인터넷 부동산"이다. 당신이 집(웹사이트)을 지으면, Vercel이 그 집을 전 세계 어디서든 방문할 수 있는 주소(URL)에 올려준다. 전기, 수도, 보안(HTTPS)까지 알아서 처리해준다. 당신은 집만 짓고, Vercel에 올리면 끝이다.

Vercel이 하는 일

순서 기능 쉽게 말하면
1자동 배포GitHub에 코드를 올리면, 자동으로 인터넷에 공개된다. 버튼 하나 누를 필요 없다
2글로벌 CDN전 세계 어디서든 빠르게 접속할 수 있도록 서버를 분산 배치한다
3자동 HTTPS보안 인증서(SSL)를 자동으로 발급한다. 브라우저에서 "안전하지 않음" 경고가 뜨지 않는다
4프리뷰 배포수정할 때마다 미리보기 URL이 자동으로 만들어진다. 확인 후 실제 반영 가능
5즉시 롤백배포 후 문제가 생기면, 이전 버전으로 즉시 되돌릴 수 있다
6커스텀 도메인"mysite.vercel.app" 대신 "mysite.com" 같은 자기만의 주소를 연결할 수 있다

배포의 흐름

바이브 코딩에서 서비스가 세상에 나오기까지의 흐름은 이렇다.

클로드 코드로 만든다 → GitHub에 올린다 → Vercel이 자동 배포한다 → URL이 생긴다 → 세상이 접속한다

이 전체 과정이 5분이면 끝난다. 과거에는 서버를 구입하고, 도메인을 등록하고, SSL 인증서를 설치하고, FTP로 파일을 올리는 데 며칠이 걸렸다. 지금은 GitHub에 Push 한 번이면 끝이다.

Vercel 요금제

순서 요금제 월 비용 특징
1Hobby무료개인 프로젝트. 대역폭 100GB/월, 빌드 6,000분/월
2Pro$20/인 (약 2.8만원)팀/비즈니스. 대역폭 1TB/월, 빌드 24,000분/월
3Enterprise맞춤 견적대규모 조직. SSO, 전담 매니저

무료

Hobby 요금제로 개인 프로젝트를 무료로 배포할 수 있다. 대부분의 바이브 코딩 프로젝트는 이 요금제로 충분하다.

Vercel 시작하기

1단계. 가입

vercel.com에 접속하여 GitHub 계정으로 회원가입한다. GitHub과 자동으로 연결된다.

2단계. 프로젝트 연결

"Add New Project"를 클릭하고, GitHub의 저장소를 선택한다. Vercel이 프로젝트 유형을 자동으로 감지한다.

3단계. 배포

"Deploy" 버튼을 클릭하면 끝이다. 1~2분 안에 URL이 생성되고, 전 세계 어디서든 접속할 수 있다.

4단계. 이후 자동 배포

한 번 연결하면, 이후에는 GitHub에 코드를 올릴 때마다 자동으로 재배포된다. 클로드 코드에게 "깃허브에 올려줘"라고 말하면, 몇 분 후 새 버전이 자동으로 반영된다.

22장. VS Code — 코드를 보고, 관리하고, 확장하는 작업대

VS Code란 무엇인가

VS Code(Visual Studio Code)는 마이크로소프트가 만든 무료 코드 편집기다. 전 세계 개발자들이 가장 많이 사용하는 개발 도구이며, 클로드 코드와 결합하면 바이브 코딩의 효율이 배가 된다.

비유하자면, VS Code는 "코드의 워드프로세서"다. MS Word가 문서를 작성하는 도구라면, VS Code는 코드를 작성하고 관리하는 도구다. 하지만 바이브 코딩에서 VS Code의 역할은 조금 다르다. 코드를 '작성'하는 것은 클로드 코드가 하고, VS Code는 그 결과물을 '확인하고 관리'하는 데 쓴다.

바이브 코딩에서 VS Code가 필요한 이유

클로드 코드만으로도 바이브 코딩은 가능하다. 하지만 VS Code를 함께 쓰면 다음과 같은 이점이 있다.

순서 이점 설명
1결과물 즉시 확인클로드가 만든 파일을 바로 열어서 내용을 확인할 수 있다. 어떤 코드가 만들어졌는지 한눈에 보인다
2실시간 미리보기Live Server 확장을 설치하면, HTML 파일을 브라우저에서 실시간으로 미리볼 수 있다
3파일 탐색프로젝트의 모든 파일과 폴더 구조를 트리 형태로 볼 수 있다
4통합 터미널VS Code 안에서 터미널을 열고 클로드 코드를 실행할 수 있다. 화면을 왔다 갔다 하지 않아도 된다
5Git 시각화코드 변경 이력을 색상으로 보여준다. 무엇이 추가되고, 삭제되고, 수정되었는지 한눈에 파악된다
6확장 프로그램수만 개의 확장으로 기능을 무한히 확장할 수 있다

VS Code 설치

설치 방법

code.visualstudio.com에서 자신의 운영체제에 맞는 버전을 다운로드하여 설치한다. Windows에서는 터미널에서 winget install Microsoft.VisualStudioCode 한 줄로도 설치할 수 있다.

VS Code 핵심 화면 구성

순서 영역 위치 역할
1Activity Bar왼쪽 끝탐색기, 검색, Git, 확장 프로그램 등으로 전환하는 아이콘 막대
2Side Bar왼쪽파일 목록, 검색 결과, Git 변경 사항 등을 보여주는 공간
3Editor중앙코드를 보고 편집하는 메인 영역. 여러 파일을 탭으로 열 수 있다
4Panel하단터미널, 출력, 문제점 등을 보여주는 하단 패널
5Status Bar맨 아래현재 파일의 언어, 인코딩, Git 브랜치 등 상태 정보 표시

바이브 코딩에서 자주 쓰는 단축키

VS Code의 단축키는 수백 개가 있지만, 바이브 코딩에서 실제로 자주 쓰는 것은 몇 개 되지 않는다.

단축키 기능 언제 쓰는가
Ctrl + `터미널 열기/닫기클로드 코드를 실행할 때
Ctrl + P파일 빠르게 열기프로젝트에서 특정 파일을 찾을 때
Ctrl + Shift + F전체 검색프로젝트 전체에서 텍스트를 찾을 때
Ctrl + B사이드바 열기/닫기편집 영역을 넓게 보고 싶을 때
Ctrl + Z실행 취소실수로 뭔가를 바꿨을 때
Ctrl + S저장파일을 수동으로 저장할 때

추천 확장 프로그램

순서 확장 프로그램 용도
1Live ServerHTML 파일을 브라우저에서 실시간 미리보기. 저장하면 자동으로 새로고침된다
2Material Icon Theme파일 종류별로 아이콘을 표시하여 구분이 쉬워진다
3GitLens코드 변경 이력을 시각적으로 보여준다. 누가, 언제, 왜 바꿨는지 확인 가능
4Prettier코드를 자동으로 정리해준다. 저장할 때 들여쓰기가 자동으로 깔끔해진다
5Auto Rename TagHTML 태그의 여는 태그를 수정하면, 닫는 태그도 자동으로 수정된다
VS Code + 클로드 코드의 조합

VS Code 안에서 터미널을 열고(Ctrl + `), 그 터미널에서 claude를 실행하면, 왼쪽에는 파일 목록, 가운데에는 코드 내용, 아래에는 클로드 코드 대화창이 한 화면에 보인다. 클로드에게 말하고 → 결과를 즉시 확인하고 → 다시 지시하는 흐름이 한 화면에서 끊김 없이 이루어진다.

도구들은 어떻게 연결되는가

5가지 도구의 역할 지도

지금까지 5가지 도구를 살펴보았다. 이 도구들은 따로 쓰는 것이 아니라, 하나의 흐름으로 연결된다.

VS Code에서 작업 → 클로드 코드에게 말로 지시 → 코드가 만들어짐 →
GitHub에 올림 → Vercel이 자동 배포 → Supabase가 데이터 관리
순서 도구 역할 비유 비용
1클로드 코드AI 개발자24시간 대기 개인 개발자월 $20~$200
2VS Code작업 환경코드의 워드프로세서무료
3GitHub코드 저장소코드의 구글 드라이브무료
4Vercel배포 플랫폼인터넷 부동산무료
5Supabase백엔드보이지 않는 사무실무료

전체 비용 계산

바이브 코딩에 드는 총 비용을 계산해보자.

필수 비용: 클로드 코드 Pro — 월 $20 (약 2.8만 원)

선택 비용: 없음 (나머지 도구 모두 무료)

총 월 비용: 약 2.8만 원

월 2.8만 원으로 AI 개발자를 고용하고, 코드를 저장하고, 인터넷에 배포하고, 데이터베이스까지 사용할 수 있다. 전문 개발자를 고용하면 월 400만~700만 원, 외주를 맡기면 프로젝트당 1,000만~5,000만 원이 드는 것과 비교하면, 이것은 혁명적인 비용 구조다.

집중 개발 기간에는 Max 요금제를 추천한다

프로젝트를 빠르게 완성하고 싶다면, 한두 달만 Max 5x ($100/월)로 올렸다가 다시 Pro로 내리면 된다. 개발 집중 기간에만 비용을 올리는 유연한 운용이 가능하다.

이제 당신은 도구를 안다

PART 1에서 직접 만들어보았고, PART 2에서 30개의 사례를 보았고, 이제 PART 3에서 도구의 정체를 이해했다. 이 순서가 중요하다. 경험 없이 이 파트를 먼저 읽었다면, "도구가 참 많네" 정도로 끝났을 것이다. 하지만 이미 써본 뒤에 읽었기 때문에, "그때 그게 이거였구나"라는 연결이 생겼을 것이다.

도구는 도구일 뿐이다. 중요한 것은 당신이 무엇을 만들 것인가다.

PART 4에서는 만든 것을 사업으로 연결하는 방법을 다룬다. 아이디어를 검증하고, MVP를 완성하고, 고도화하고, 투자를 유치하는 흐름이다. 도구를 이해한 지금, 비로소 사업 이야기를 할 준비가 되었다.

도구를 알면 두려움이 사라진다.
두려움이 사라지면 실행이 시작된다.
실행이 시작되면, 사업이 만들어진다.
